1.4 Getting Refer Installed

REFER can be downloaded from https://github.com/REFER-SBM/REFER/ site. The installation package includes a setup assistant that automatically installs the executable software, the necessary configuration files, and a collection of sample projects.

REFER can also be compiled and linked from its source code. The source code is written in C++(1) and provided as a Microsoft Visual Studio Solution that leverages Microsoft Foundation Classes (MFC(2)) to build the interactive environment. It can be compiled and linked using Microsoft Visual Studio 2008 ((C) Microsoft Corporation 2007).

The solution can be exported to Microsoft Visual Studio Community 2015 or 2019. The warnings generated during the migration process can be safely ignored(3) .

In both cases, the terms of use are similar—though not identical—to those of copyleft licenses (in particular, the source code is not bundled with the executable). You can study, use, distribute and modify it, as long as the software and any derivates remain free and open source. You cannot claim ownership of the software or pass it off as your own original work. If you use it for academic purposes, please cite this handbook, the REGEO group, or the corresponding reference for the specific module or algorithm.

This help document is provided with the application. It can be accessed through the "help" command located at the end of the main toolbar. Alternatively, the help file can be opened directly by running Refer.chm, located in the Help folder.

The help document was created using Microsoft's HTML Help Workshop compiler (R). The file Refer.hhp is also provided for users interested in editing the help content.



NOTES:

(1) For readers interested in learning C++, we recommend "Eckel B. Thinking in C++, Second edition, Prentice Hall, 2000".

(2) Microsoft Foundation Class Library (MFC) is a C++ object-oriented library for developing desktop applications for Windows. For those interested in learning MFC, we recommend "Jones R.M. Introduction to MFC Programming with Visual C++, Prentice Hall, 2000".

(3) The application uses two different approaches to produce a Vectorized Line-Drawing. One of these approaches is a third-party contribution:

Fonseca M.J., Pimentel C. and Jorge J.A. (2002) CALI: An Online Scribble Recognizer for Calligraphic Interface. AAAI Technical Report SS-02-08. Compilation copyright © 2002, AAAI (www.aaai.org)

To compile the application with an alternative to Microsoft Visual Studio 2008, the CALI library must be rebuilt using Microsoft Visual Studio Community 2015 or 2019, or similar compiler and linker: Open and run the Recognizer.sln file located in the RECOGNIZER folder. Once the resulting library (Recognizer.lib) is generated, copy it to the "lib" folder, where the freeglut lib is located.
